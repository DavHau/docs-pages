System Requirements

Hardware:

  • 16GB+ RAM (including swap)
  • 4+ CPU (6+ CPU recommended)
  • 30GB+ available disk space
  • Internet connection

Installed software:

  • nix-shell (click link for instructions)
  • Xcode developer tools (mac only, installed from app store)

We use the NixOS package manager to develop Holochain core and apps.
NixOS is the operating system used by HoloPorts and…
Nix tools work natively across Mac OS X and Linux.

The main nix tools used in Holochain development are:

  • nix-shell: a managed bash shell
  • nix-env: a cross-platform package manager
It is convenient that we can provide standard development tools and packages with a single tool across many operating systems.

While working on Holochain you will usually have a nix shell active.
When you close the shell all the extra dependencies will be cleaned up.

Install nix shell

If you did not already install nix shell do that now.
Open a command line terminal and run:

curl https://nixos.org/nix/install | sh

Follow any additional instructions at the end of the installation process.

Enter the nix shell

Nix is configured by default.nix files.
These set all dependencies and variables added to your terminal.
There is a master default.nix file updated from time to time.
You can extend this or use it directly from https://holochain.love.
Opening a shell from holochain.love downloads updates automatically.

To enter a nix shell environment open a new terminal window and run:

nix-shell https://holochain.love

The first try will take some time to download.
It is much faster after the initial download.
You will see a neon green prompt starting with:

[nix-shell:

Try the holochain tools

In the nix shell you can access Holochain dev tools:

  • hc: A general purpose app development toolkit.
  • holochain: Runs holochain apps.

You can see both are installed by checking their versions.
Inside a nix shell run:

hc --version

holochain --version

Exit the nix shell

Nix cleans up everything it adds when closed.
You need to re-enter the shell each time you want to work.
To close nix shell run this inside the shell:

exit

Permanently install holochain tools (optional)

nix-env can be used to permanently install holochain tools.
This means that hc and holochain can be run at any time.
nix-env can also uninstall tools it installs.

nix-env installations override nix-shell.
Use nix-shell if you want the latest releases.
Use nix-env if you want a single copy of the tools everywhere.

To install holochain and hc with nix-env run (on one line):

nix-env -f https://holochain.love -iA holochain.holochain holochain.hc

Test this by running the following outside nix-shell:

hc --version

holochain --version

To uninstall holochain and hc with nix-env run:

nix-env -e holochain hc

System Requirements

Hardware:

  • 8GB+ RAM (16GB+ recommended)
  • 4+ CPU (6+ CPU recommended)
  • 30GB+ available disk space
  • Internet connection

Operating System & Software:

  • Windows 8+
  • Powershell 2.0+
  • git installed
  • Vagrant
  • VirtualBox
  • Vagrant nixos plugin

To develop with Holochain, a managed work environment is provided for your machine that is guaranteed to have the required dependencies. It’s called a “nix-shell.” To develop, you will want to work “inside” the nix-shell, which is a wrapper around your normal shell that comes with all the requirements built-in, and leaves your normal shell untampered with for all your other development needs. In the case of Windows, as nix-shell is not native to it, this requires also working inside a Virtual Machine, "Vagrant".

OPen a command line terminal, and run each of the following commands, in order:

  1. Clone the holochain repository, in order to access the Holochain "nix-shell":

    git clone https://github.com/holochain/holochain-rust.git

  2. Enter the new holochain-rust directory:

    cd holochain-rust

  3. Install a required vagrant plugin:

    vagrant plugin install vagrant-disksize

  4. Set up and configure the virtual machine:

    vagrant up

  5. Enter a "secure shell" SSH session in the virtual machine:

    vagrant ssh

  6. Change directories into the holochain-rust folder:

    cd /vagrant

  7. Enter the nix-shell environment:

    nix-shell

    This can take a moderately long time the first time, depending on network and computer speed. It takes less time in future cases. Your shell context should become neon green and begin with "[nix-shell:".

  8. Test that you can access the tools:

    hc --version

    You should see a version number that represents the version of Holochain that you're using.

Anytime that would like to use the Holochain command line tools, you will want to initialize a nix-shell session first, which will require you to perform steps 2, 5, 6, and 7 again. When you are done, and wish to exit nix-shell, just run the following command:

exit

Since Vagrant is also a shell, you will want to exit it as well, by running the following:

exit

Starter Code For Your First App

To generate starter code in a new folder for a Holochain app, run:

hc init my_first_app

In the new "my_first_app" folder is the basic structure of a new Holochain project. Open it up in a file browser or text editor to start to check it out.

Once you've completed that, you will be able to generate a new "Zome," the basic module format for Holochain.

To try it, make sure your terminal is set to the directory of "my_first_app" by running

cd my_first_app

and then run:

hc generate zomes/my_zome

This will add some starter code into the "zomes/my_zome" sub folder, which you can open up in a text editor and start checking out!

To run the tests (found in the “my_first_app/test/index.js” file), run the following command:

hc test

Otherwise, to familiarize with the tools, run the following command:

hc help
