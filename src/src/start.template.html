System Requirements

Hardware:

  • 16GB+ RAM minimum

Software:

  • nix-shell (instructions included below)

To develop with Holochain, a managed work environment is provided for your machine that is guaranteed to have the required dependencies. It’s called a “nix-shell.” To develop, you will want to work “inside” the nix-shell, which is a wrapper around your normal shell that comes with all the requirements built-in, and leaves your normal shell untampered with for all your other development needs.

Open a command line terminal and run each of the following commands, in order:

  1. Install nix:

    curl https://nixos.org/nix/install | sh

    Follow any additional instructions noted towards the end of the logs from the installation process.

  2. Enter the nix-shell environment:

    nix-shell https://holochain.love

    This can take a moderately long time the first time, depending on network and computer speed. It takes less time in future cases. Your shell context should become neon green and begin with "[nix-shell:".

  3. Test that you can access the tools:

    hc --version

    You should see a version number that represents the version of Holochain that you're using.

Anytime that would like to use the Holochain command line tools, you will want to initialize a nix-shell session first, which will require you to perform step 2 again. When you are done, and wish to exit nix-shell, just run the following command:

exit

System Requirements

Hardware:

  • 8GB+ RAM (16GB+ recommended)
  • 4+ CPU (6+ CPU recommended)
  • 30GB+ available disk space
  • Internet connection

Operating System & Software:

  • Windows 8+
  • git
  • Vagrant
  • VirtualBox

To develop with Holochain, a managed work environment is provided for your machine that is guaranteed to have the required dependencies. It’s called a “nix-shell.” To develop, you will want to work “inside” the nix-shell, which is a wrapper around your normal shell that comes with all the requirements built-in, and leaves your normal shell untampered with for all your other development needs. In the case of Windows, as nix-shell is not native to it, this requires also working inside a Virtual Machine, "Vagrant".

OPen a command line terminal, and run each of the following commands, in order:

  1. Clone the holochain repository, in order to access the Holochain "nix-shell":

    git clone https://github.com/holochain/holochain-rust.git

  2. Enter the new holochain-rust directory:

    cd holochain-rust

  3. Install a required vagrant plugin:

    vagrant plugin install vagrant-disksize

  4. Set up and configure the virtual machine:

    vagrant up

  5. Enter a "secure shell" SSH session in the virtual machine:

    vagrant ssh

  6. Change directories into the holochain-rust folder:

    cd /vagrant

  7. Enter the nix-shell environment:

    nix-shell

    This can take a moderately long time the first time, depending on network and computer speed. It takes less time in future cases. Your shell context should become neon green and begin with "[nix-shell:".

  8. Test that you can access the tools:

    hc --version

    You should see a version number that represents the version of Holochain that you're using.

Anytime that would like to use the Holochain command line tools, you will want to initialize a nix-shell session first, which will require you to perform steps 2, 5, 6, and 7 again. When you are done, and wish to exit nix-shell, just run the following command:

exit

Since Vagrant is also a shell, you will want to exit it as well, by running the following:

exit

Starter Code For Your First App

To generate starter code in a new folder for a Holochain app, run:

hc init my_first_app

In the new "my_first_app" folder is the basic structure of a new Holochain project. Open it up in a file browser or text editor to start to check it out.

Once you've completed that, you will be able to generate a new "Zome," the basic module format for Holochain.

To try it, make sure your terminal is set to the directory of "my_first_app" by running

cd my_first_app

and then run:

hc generate zomes/my_zome

This will add some starter code into the "zomes/my_zome" sub folder, which you can open up in a text editor and start checking out!

To run the tests (found in the “my_first_app/test/index.js” file), run the following command:

hc test

Otherwise, to familiarize with the tools, run the following command:

hc help
